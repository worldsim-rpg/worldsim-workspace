worldsim — руководство разработчика

Для тех, кто не пишет код руками, но хочет через Claude Code добавлять фичи в игру без того, чтобы всё разнести. Читать сверху вниз. Длина — ~20 минут.

0. За 2 минуты: как это устроено

Игра worldsim — 7 независимых git-репо. Один из них — worldsim-workspace — это ваша рабочая папка с документацией и общими пакетами. Остальные шесть — отдельные агенты (мозги игры): оркестратор, мир-движок, хранитель канона, рассказчик, диалог-NPC, прогрессия игрока.

Агенты общаются между собой не по сети, а как обычные pip-пакеты внутри одного процесса. Это максимально просто: у вас не будет ни контейнеров, ни MCP, ни очередей.

Архитектура целиком — architecture-guide.html. Сначала прочитайте её, потом возвращайтесь сюда.

1. Первый запуск

1.1. Клонируйте 7 репо рядом

mkdir worldsim
cd worldsim
git clone https://github.com/b3axap/worldsim-workspace
git clone https://github.com/b3axap/worldsim-orchestrator
git clone https://github.com/b3axap/worldsim-world-builder
git clone https://github.com/b3axap/worldsim-canon-keeper
git clone https://github.com/b3axap/worldsim-scene-master
git clone https://github.com/b3axap/worldsim-npc-mind
git clone https://github.com/b3axap/worldsim-personal-progression

Все семь должны лежать на одном уровне — sync-скрипт ищет их как соседние папки.

1.2. Python 3.11+, venv

cd worldsim-workspace
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

1.3. Установка

# общие пакеты
pip install -e packages/schemas
pip install -e packages/prompts

# агенты
pip install -e ../worldsim-world-builder
pip install -e ../worldsim-canon-keeper
pip install -e ../worldsim-scene-master
pip install -e ../worldsim-npc-mind
pip install -e ../worldsim-personal-progression

# оркестратор — последним (он ставит typer, rich, anthropic и подтягивает всех)
pip install -e ../worldsim-orchestrator

1.4. API-ключ

cp ../worldsim-orchestrator/.env.example ../worldsim-orchestrator/.env
# отредактируйте .env — впишите ANTHROPIC_API_KEY

1.5. Запуск

worldsim new                    # создать новый мир (интерактивно)
worldsim play --world <id>      # играть в выбранный мир
worldsim worlds                 # список миров

2. Куда вы смотрите, когда что-то не работает

СимптомКуда идти
Игра не стартует, ImportErrorНе все 6 агентов + 2 пакета установлены через pip install -e. Проверьте активный venv.
"ANTHROPIC_API_KEY not set"worldsim-orchestrator/.env не создан или не подхватился.
Игра повторяет "не могу, потому что X"Hard-validator срабатывает. Логика — worldsim-orchestrator/src/worldsim_orchestrator/validators.py.
NPC знает то, чего знать не долженПроблема в worldsim-npc-mind/prompts/dialog.md или в том, что его knowledge при init заполнен слишком щедро.
Сцены сухие / слишком длинныеworldsim-scene-master/prompts/render.md.
Игрок бесконечно качает скиллПороги в worldsim-personal-progression/prompts/progression.md.
LLM возвращает мусор, падает JSON-парсСмотрите WORLDSIM_DEBUG=1: в stderr будет raw-выход модели.

3. Железное правило: где править

90% изменений — это правка markdown-промптов, а не Python. Если ваш Claude Code хочет залезть в .py файл, первый вопрос — точно ли это не промпт?

Код в src/.../agent.py у каждого агента — это тонкий клей: “возьми input, подставь в промпт, спарси JSON”. Поведение живёт в prompts/*.md.

4. Как добавить фичу, не сломав всё

4.1. Определите, в чью зону попадает изменение

Откройте docs/agent-map.md. Там таблица: кто что пишет в канон. Ваша фича должна полностью лежать в одной зоне. Если тянет в несколько — разделите на части.

4.2. Проверьте, не задевает ли это канон (схемы)

Если нужно новое поле в сущности (например, у Character появляется mood):

  1. Правка в worldsim-workspace/packages/schemas/src/worldsim_schemas/schemas.py.
  2. Запустите python -m pytest -q в packages/schemas/.
  3. ./repo-setup/sync-all.sh — это разольёт новые схемы во все агент-репо как _schemas/.
  4. В каждом репо, где agent.py читает/пишет это поле — добавьте поддержку.
  5. Промпт, который теперь должен генерировать это поле — правка prompts/*.md.
Никогда не правьте _schemas/ внутри агент-репо руками. Это снимок. Меняется только в workspace и раскатывается sync-скриптом.

4.3. Новая hard-constraint

Живут в worldsim-orchestrator/src/worldsim_orchestrator/validators.py. Каждая — маленькая функция, возвращает ConstraintResult(ok, code, message). Добавьте вызов в validate_intent.

Тесты — обязательно, в worldsim-orchestrator/tests/test_validators.py. Пример на каждый позитив и каждый негатив.

4.4. Новая soft-характеристика / новое поведение NPC

Это чисто промпты. Не пишите код.

  1. Найдите промпт, который это генерирует (обычно world_init.md или turn_update.md).
  2. Добавьте в него раздел “Правила” с новым пунктом.
  3. Если поле новое — пункт 4.2 выше.
  4. Запустите игру, создайте новый мир, посмотрите, как получается.

5. Что запрещено

  • Один файл канона — один писатель. Если вам кажется, что world-builder должен записать что-то в player_progression — это всегда ошибка. Добавьте патч в зоне personal-progression.
  • Не увеличивайте количество агентов без веской причины. Сейчас их 6. Каждый новый — это новый сетевой контакт, новый источник расхождений. Если фича укладывается в существующий агент — кладите туда.
  • Не ломайте monотонность инвариантов. progress, known_facts, Secret.status, condition=dead — только в одну сторону. Это поддерживает consistency-проверки.
  • Не меняйте схему сейва без бэк-совместимости. Если у пользователя есть сохранения — они не должны умирать после git pull.

6. Поток работы через Claude Code

Простой ритм для non-coder:

  1. Сначала опишите Claude, что хотите. Не “исправь такой-то файл”, а “хочу, чтобы NPC запоминали, когда игрок им врёт, и снижали attitude в повторных встречах”.
  2. Попросите сначала предложить план. Не сразу писать код.
  3. План должен упомянуть: какой агент задействован, какие промпты правятся, нужна ли правка схем, какие тесты добавляются.
  4. Если план касается больше одного агента — насторожитесь. Большинство корректных правок — одно-агентные.
  5. Сохраните план как issue в нужном репо, потом правьте.
  6. После правки — sync-all.sh (если меняли схемы), python -m pytest -q во всех задетых репо.
  7. Запустите игру, создайте новый мир (не старый сейв!), проверьте вживую.

7. Как безопасно обновляться

# в корне worldsim/
for d in worldsim-*; do
  (cd "$d" && git pull --ff-only)
done

# если менялась схема — workspace раскатит свежие _schemas/_prompts
cd worldsim-workspace
./repo-setup/sync-all.sh

После этого перезапустите тесты. Если что-то упало — смотрите последний git log в workspace, там обычно причина.
