worldsim — скиллы Claude Code

Быстрый справочник: что такое скиллы, как установить и когда вызывать каждый из них.

Что такое скиллы

Скиллы — это команды Claude Code, заточенные под конкретные задачи проекта. Вместо того чтобы каждый раз объяснять Claude «синхронизируй схемы из workspace во все репо», достаточно написать /schema-sync.

Технически скилл — это файл SKILL.md с инструкцией. Claude читает её и выполняет задачу. Все скиллы worldsim хранятся в worldsim-workspace/skills/ и устанавливаются одним скриптом.

Установка

cd repos/worldsim-workspace
bash skills/install.sh

Скрипт найдёт папку Claude Code на вашей машине (Windows/macOS/Linux) и скопирует туда все скиллы. После этого перезапустите Claude Code.

Проверка после установки: напишите в Claude Code /check-setup — он покажет что установлено правильно, а что нет.
После обновления скиллов (например, после git pull в worldsim-workspace) повторно запустите install.sh и перезапустите Claude Code.

Подробнее по каждому скиллу

/check-setup setup · audit

Двухрежимный скилл.

Setup mode — если репо не склонированы или venv отсутствует. Claude проходит по шагам: структура папок → клонирование 7 репо → создание venv → pip install → schema-sync → установка скиллов → API ключ. Каждый шаг с подтверждением.

Audit mode — если окружение уже есть. Проверяет всё и выводит таблицу:

  • Структура папок (sandbox не является git-репо, все 7 репо на месте)
  • Git remotes настроены правильно
  • Python venv, все пакеты установлены
  • ANTHROPIC_API_KEY задан
  • Схемы свежие (не старше 7 дней)
  • Скиллы установлены
  • CLI: git, python, gh

После отчёта предлагает починить механические проблемы (pip install, schema-sync, install.sh) — сами, без вашего участия.

Примеры:
первый запуск, помоги настроить
/check-setup
проверь что всё ок
/ws-status git · overview

Запрашивает состояние всех 7 репо и выводит одну компактную таблицу. Видно сразу: в каком репо есть незапушенные коммиты, где отстали от remote, где есть несохранённые изменения.

После таблицы — одна строка-подсказка: нужен ли /github-sync или /schema-sync.

Примеры:
/ws-status
что изменилось в репо
покажи статус
/github-sync git · sync

Синхронизирует все репо с GitHub. Если направление не указано — спрашивает: pull, push или оба.

Pull использует --rebase --autostash, поэтому незакоммиченные изменения не потеряются — git сам их спрячет и вернёт после rebase.

При конфликтах останавливается и описывает проблему — не пытается разрешать автоматически.

Примеры:
/github-sync
запулли все репо
запуш worldsim-orchestrator
/schema-sync schemas · sync

Запускает repo-setup/sync-all.sh из workspace. Скрипт берёт содержимое packages/schemas/ и packages/prompts/ и копирует его в _schemas/ и _prompts/ каждого агент-репо.

После синхронизации показывает git diff --stat по каждому репо — видно что реально изменилось. Напоминает запустить тесты.

Никогда не редактируйте _schemas/ и _prompts/ в агент-репо вручную — они перезапишутся при следующем schema-sync.
Примеры:
/schema-sync
синхронизируй схемы
обнови _schemas во всех агентах
/run-tests tests · pytest

Запускает pytest параллельно во всех агент-репо (и в packages/schemas, packages/prompts workspace). Сводная таблица: ✓ passed / ✗ FAILED + количество тестов и время.

Для упавших тестов выводит --tb=short прямо в ответ.

Можно указать конкретный агент: прогони тесты в worldsim-orchestrator.

Примеры:
/run-tests
запусти тесты
прогони тесты в worldsim-canon-keeper
/scaffold-agent scaffold · new agent

Создаёт новый агент-репо из шаблонов workspace. Спрашивает три вещи: имя репо (worldsim-*), Python-имя пакета и однострочное описание.

Что делает автоматически:

  1. Запускает generate-scaffold.sh — создаёт структуру с README, CLAUDE.md, pyproject.toml, src/
  2. Добавляет репо в agents.txt
  3. Запускает schema-sync чтобы новый агент сразу получил схемы
  4. Делает первый git commit
  5. Предлагает создать репо на GitHub через gh
Примеры:
/scaffold-agent
создай нового агента plot-weaver
новый агент для управления погодой

Типичный рабочий день

# Утром — смотрим состояние и обновляемся
/ws-status
/github-sync   # pull

# Редактируем схемы в packages/schemas/ ...

# Раскатываем изменения в агенты
/schema-sync

# Проверяем что ничего не сломалось
/run-tests

# Пушим
/github-sync   # push

Обновление скиллов

Скиллы живут в worldsim-workspace/skills/ и обновляются вместе с репо. После git pull в workspace:

bash skills/install.sh

Перезапустите Claude Code — новые версии подтянутся автоматически.
